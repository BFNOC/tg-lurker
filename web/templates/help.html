使用指南

如何配置和使用 tg-lurker

快速开始

tg-lurker 是一个 Telegram 群聊监控工具，它以 Userbot（真人账号）方式静默加入群聊，收集消息并通过 AI 生成每日摘要。

第一步：获取 Telegram API 凭证
  1. 访问 https://my.telegram.org
  2. 使用你的 Telegram 手机号登录
  3. 点击 API development tools
  4. 填写应用信息（App title 和 Short name 随意填写）
  5. 获取 API_ID（数字）和 API_HASH（字符串）
请使用小号或备用号。频繁操作可能触发 Telegram 风控。建议新号先正常使用几天再运行脚本。
第二步：获取 Owner ID

Owner ID 是接收摘要推送的用户 ID。获取方式：

  1. 在 Telegram 中搜索 @userinfobot
  2. 向它发送任意消息
  3. 它会回复你的 User ID（纯数字）

或者使用 @RawDataBot，转发任意消息给它即可看到 sender ID。

第三步：配置 LLM API

支持任何 OpenAI 兼容的 API。推荐国产平台（便宜）：

平台Base URL模型价格
DeepSeek https://api.deepseek.com/v1 deepseek-chat ¥1/M tokens
通义千问 https://dashscope.aliyuncs.com/compatible-mode/v1 qwen-plus ¥0.8/M tokens
智谱 AI https://open.bigmodel.cn/api/paas/v4 glm-4-flash 有免费额度

20 个群每日摘要成本约 ¥0.05，几乎免费。

第四步：首次登录

Telethon 首次运行需要交互式验证：

# 本地运行（Docker 环境）
docker compose run --rm tg-lurker python main.py

# 按提示输入：
# 1. 手机号（带国际区号，如 +86xxxxx）
# 2. Telegram 发送的验证码
# 3. 如有两步验证，输入密码

# 登录成功后 session 文件保存在 ./data/
# 之后正常启动：
docker compose up -d
代理配置

如果服务器无法直连 Telegram，需要配置代理：

环境变量说明示例
PROXY_TYPE代理类型socks5 / http
PROXY_HOST代理地址127.0.0.1
PROXY_PORT代理端口1080
LLM_PROXY_URLLLM 调用代理（独立）http://127.0.0.1:7897
Telegram 代理和 LLM 代理是独立配置的。Docker 中使用 host.docker.internal 访问宿主机代理。
常见问题
如何添加监控群组？

用你的 Telegram 账号加入群组即可。机器人启动后会自动同步已加入的群列表，然后在「群组」页面勾选需要监控的群。

摘要生成失败怎么办？

检查 LLM 配置是否正确（Base URL、API Key、Model）。可以在设置页面修改后，到仪表盘手动触发测试。失败的群消息不会被清空，下次会重试。

账号会被封吗？

风险存在但较低。建议：1) 使用备用号；2) 新号先正常使用几天；3) 不要频繁加入/退出群；4) 不要主动发消息。本工具只做被动监听。

数据安全吗？

所有数据存储在本地 SQLite 中，不上传任何第三方服务（除 LLM API 调用）。Session 文件等同于登录凭证，请妥善保管。
